# 快速开始

## 环境准备

- Node.js `24.15.0`
- pnpm `10.33.0`

## 安装依赖

```bash
pnpm install
```

## 本地开发

```bash
# 启动 packages/tools 的 watch
pnpm run dev:packages

# 启动 apps 的开发进程
pnpm run dev:apps

# 一键启动（默认命令）
pnpm run dev
```

## 构建与校验

```bash
pnpm run build
pnpm run lint
pnpm run typecheck
pnpm run test
pnpm run e2e
```

## 常见场景命令

| 场景 | 命令 |
| --- | --- |
| 格式化检查 | `pnpm run format` |
| 创建版本变更记录 | `pnpm run changeset` |
| 计算并写入版本号 | `pnpm run version-packages` |
| 发布包 | `pnpm run release` |
| 清理构建产物与依赖目录 | `pnpm run clean` |

## 开发建议

1. 先在 `packages/*` 完成可复用能力，再在 `apps/*` 接入。
2. 变更公共 API 时同步补充测试与 Wiki。
3. 发布前至少执行一次 `build + test + typecheck`。

## 常见问题

**Q：`pnpm install` 后执行 `pnpm run dev` 报错找不到模块？**

先执行一次 `pnpm run build` 让各 `packages/*` 生成 `dist`，再启动 `dev`。`dev:packages` 会以 watch 模式持续编译，但首次运行前需要存在产物。

**Q：`pnpm run dev` 启动后浏览器插件没有更新？**

浏览器插件需要在 `chrome://extensions` 页面手动点击"刷新"后才会加载最新产物，Vite HMR 不会自动触发插件重载。

**Q：VS Code 插件本地调试怎么启动？**

在 VS Code 中打开仓库，按 `F5` 启动扩展开发宿主（Extension Development Host），然后在新窗口中执行命令 `Scribdown: Open Preview`。

**Q：`pnpm run format` 报格式错误怎么修？**

执行 `pnpm exec prettier --write .` 自动修复格式，然后重新运行 `pnpm run format` 确认通过。

**Q：`pnpm run typecheck` 报找不到类型声明？**

确认依赖包已构建（`pnpm run build`），各 `packages/*/dist` 目录存在后再重试。

**Q：`pnpm run e2e` 报找不到浏览器？**

首次运行前需要安装 Playwright 浏览器：

```bash
pnpm exec playwright install
```
