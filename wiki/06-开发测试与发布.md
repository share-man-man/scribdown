# 开发测试与发布

## 质量保障命令

| 目标 | 命令 | 说明 |
| --- | --- | --- |
| 代码检查 | `pnpm run lint` | 基于 ESLint 执行静态检查 |
| 代码格式检查 | `pnpm run format` | 基于 Prettier 校验格式 |
| 类型检查 | `pnpm run typecheck` | 全工作区 TypeScript 类型检查 |
| 单元测试 | `pnpm run test` | Turborepo 调度各包测试 |
| 端到端测试 | `pnpm run e2e` | Playwright 执行 `tests/e2e` |

## 当前测试现状

- 全局 Vitest 已配置 `**/*.test.ts` 与 `**/*.test.tsx` 匹配规则。
- `packages/shared/src/constants.test.ts` 已包含基础常量测试。
- Playwright 配置已就绪，`tests/e2e` 目录待补业务用例。

## Turborepo 任务编排

- `dev`：不缓存，持久运行。
- `build`：依赖上游 `^build`，产物目录 `dist/**` 与 `build/**`。
- `test`：依赖上游 `^build`，覆盖率目录 `coverage/**`。
- `typecheck`：依赖上游 `^typecheck`。

## 版本管理（Changesets）

推荐流程：

1. 开发完成后执行 `pnpm run changeset` 记录变更。
2. 在主分支发布前执行 `pnpm run version-packages` 计算版本并更新 changelog。
3. 通过 CI 或发布机执行 `pnpm run release` 发布。

## 发布前检查清单

- 关键功能变更已补单元测试或 E2E 测试
- `lint`、`typecheck`、`test` 全部通过
- 影响用户行为的变更已写入 changeset
- Wiki/README 已同步更新
