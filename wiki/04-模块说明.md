# 模块说明

## apps/browser-extension

- 入口：`src/main.tsx`
- 页面：`src/App.tsx`
- 样式：`src/styles.css`
- 清单：`public/manifest.json`

职责说明：
- 挂载 React 应用并展示浏览器插件预览面板。
- 使用 `@scribdown/markdown-renderer` 将 Markdown 渲染为 HTML。
- 使用 `@scribdown/ui-handdrawn` 输出统一手绘卡片风格。

## apps/vscode-extension

- 入口：`src/extension.ts`
- 命令 ID：`scribdown.openPreview`

职责说明：
- 激活时注册预览命令。
- 命令触发后创建 Webview 并渲染基础预览 HTML。
- 为后续接入完整 markdown-renderer 预留扩展点。

## packages/markdown-renderer

- 入口：`src/index.ts`
- 关键导出：
  - `renderMarkdown(markdownText, options)`
  - `renderCodeToHtml(codeText, language)`
  - `sanitizeHtmlWithDomPurify(unsafeHtml)`

职责说明：
- 承载 Markdown 到 HTML 的统一转换能力。
- 提供可选的结构化与字符串级双层清洗能力。
- 提供代码高亮 HTML 输出能力。

## packages/ui-handdrawn

- 入口：`src/index.tsx`
- 样式：`src/styles.css`
- 关键组件：`HanddrawnCard`

职责说明：
- 提供统一手绘视觉卡片容器。
- 使用 `@scribdown/shared` 的项目名和主题常量，保持跨端一致。

## packages/shared

- 常量：`src/constants.ts`
- 枚举：`src/enums.ts`
- 导出汇总：`src/index.ts`

职责说明：
- 统一项目级命名、平台类型、主题类型和命令常量。
- 避免多处硬编码，降低跨端文案与标识分叉风险。

## tools

- 说明文件：`tools/README.md`

职责说明：
- 存放脚本、自动化流程与开发辅助工具。
