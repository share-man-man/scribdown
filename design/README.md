# Scribdown UI 设计文档

## 文档结构

当前设计文档收敛为两份主规范：

| 分类 | 文件 | 内容 |
| --- | --- | --- |
| 主规范 | [spec/ui-spec.md](./spec/ui-spec.md) | 产品概览、信息架构、视觉基础、渲染模型、状态与交互、画板范围 |
| 组件规范 | [spec/component-spec.md](./spec/component-spec.md) | 公开视觉组件、共享文本流层、组件结构与状态规格 |

## 阅读顺序

建议按下面顺序查阅：

1. 先看 [spec/ui-spec.md](./spec/ui-spec.md)，建立整体产品与页面级认知
2. 再看 [spec/component-spec.md](./spec/component-spec.md)，确认组件职责与验收边界
3. 需要出图时查看 [guides/pencil-guide.md](./guides/pencil-guide.md)
4. 需要统一内容样例时查看 [fixtures/markdown-fixture.md](./fixtures/markdown-fixture.md)
